Email & Password

Quick Start
Minimal setup,
maximum security

Three files. Full auth. OAuth + credentials, sessions, RBAC — ready in minutes.

server.ts 
import {
  createReactAuth,
  createGoogleProvider,
  migrate,
} from "@1moby/just-auth";

// Run migrations (non-destructive)
await migrate(db);

const auth = createReactAuth({
  providers: [
    createGoogleProvider({
      clientId: env.GOOGLE_ID,
      clientSecret: env.GOOGLE_SECRET,
      redirectURI: "/api/auth/callback/google",
    }),
  ],
  database: db,
  credentials: true,
});

// Handle all auth routes
const res = await auth.handleRequest(request);
if (res) return res;
App.tsx 
import {
  SessionProvider,
  useSession,
  signIn,
  signOut,
} from "@1moby/just-auth/client";

function App() {
  return (
    <SessionProvider>
      <Profile />
    </SessionProvider>
  );
}

function Profile() {
  const { data, status } = useSession();

  if (status === "loading") return <p>Loading...</p>;
  if (!data) return (
    <button onClick={() => signIn("google")}>
      Sign In
    </button>
  );

  return <p>Hello, {data.user.name}</p>;
}
Security
Audited & hardened

285 tests including 90 dedicated security tests. Every vulnerability class covered with regression tests.

✓ CSRF Protection

Origin header verification on all state-changing endpoints. Referer fallback for legacy browsers. SameSite=Lax cookies as primary defense.

✓ Timing-Safe Comparisons

SHA-256 normalized constant-time comparison for OAuth state. PBKDF2 constant-time verification. Dummy hash for non-existent users prevents timing oracles.

✓ Injection Prevention

All queries use parameterized placeholders. Table prefix validated with strict regex. Tested against 10+ SQL injection payloads.

✓ Open Redirect Blocked

Same-origin validation on all redirect URLs. Blocks javascript:, data:, protocol-relative, and domain-spoofing variants.

✓ XSS Prevention

HTML attribute escaping. Script context </ sequence escaping. Content-Security-Policy headers on all HTML responses. X-Frame-Options: DENY.

✓ Enumeration Resistance

Identical error responses for valid/invalid users. Generic registration errors. No email existence disclosure. Constant-time code paths.
